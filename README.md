# 设计模式演示项目说明

## 一、目标受众

| 人群 | 特征 |
|------|------|
| **计算机专业学生** | 正在学习《软件设计》或《设计模式》课程，需要理解抽象概念 |
| **Java 初学者** | 有一定编程基础，想从理论走向实践的开发者 |
| **面试备考者** | 需要快速回顾设计模式核心思想和代码实现 |
| **技术讲师** | 需要课堂演示素材，让抽象概念可视化 |

## 二、核心痛点与解决方式

| 痛点 | 本项目提供的解决方案 |
|------|-------------------|
| 设计模式太抽象，只看代码难理解 | 每个模式配有**交互式 HTML 演示页面**（如观察者模式的气象站可实时调节参数） |
| 缺乏可运行的完整代码 | 每个模式都有**独立的 Java 实现**，可直接编译运行 |
| 网上资料零散，不成体系 | 同一项目内覆盖 **11 种经典设计模式**，结构统一 |
| 不知道在什么场景下使用 | 每个模式基于**真实业务场景**命名（如电商订单、文件系统遍历） |

## 三、为什么非用这个工具不可？

1. **"看得见"的设计模式**：比起纯文字或静态代码，通过滑块调节气压、温度等参数，实时看到观察者面板的变化，理解成本大幅降低。
2. **"学得会"的代码结构**：项目结构清晰，每个模式独立成包，从简单到复杂都有对应实现（如工厂模式分简单/方法/抽象三种）。
3. **"拿得走"的课堂作业**：学生可直接基于现有代码修改或扩展，完成课程设计。

## 四、使用场景

- **课堂教学**：老师在投影上演示，学生直观看到观察者模式的"发布-订阅"机制。
- **课后自学**：学生对照 HTML 页面和 Java 代码，理解模式原理。
- **面试速查**：快速定位到某个模式的典型实现，复习核心类图关系。
- **课程设计基础**：基于现有项目扩展新功能，减少从零开始的成本。

## 五、技术栈

- Java 17
- Maven
- HTML / CSS / JavaScript（演示页面）

## 六、快速启动

```bash
# 编译 Java 代码
mvn compile

# 查看演示页面
# 直接用浏览器打开 docs 目录下的 .html 文件即可
```

## 七、项目结构

```
docs/          -- 各设计模式的交互式 HTML 演示页面
src/main/java/designpatterns/  -- 各设计模式的 Java 实现
  ├── adapter/         -- 适配器模式
  ├── chain/           -- 责任链模式
  ├── command/         -- 命令模式
  ├── composite/       -- 组合模式
  ├── decorator/       -- 装饰器模式
  ├── facade/          -- 外观模式
  ├── factory/         -- 工厂模式（简单/方法/抽象）
  ├── iterator/        -- 迭代器模式
  ├── observer/        -- 观察者模式
  ├── singleton/       -- 单例模式
  └── state/           -- 状态模式
```
